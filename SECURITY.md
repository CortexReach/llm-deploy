# Security

## Reporting Security Issues

Please do not open public issues for security-sensitive reports.

Report suspected vulnerabilities through GitHub's private vulnerability
reporting flow for this repository, if enabled by the organization.

## Sensitive Data Policy

Public examples in this repository must not include:

- Credentials, tokens, API keys, or cookies
- Private hostnames, internal IP addresses, or VPN details
- SSH users, ports, or bastion details
- Customer, patient, employee, or user data
- Production logs, traces, metrics endpoints, or dashboard URLs
- Proprietary prompts or workflow-specific sensitive examples

## Deployment Security

The examples are reference configurations. They do not replace authentication,
network access control, rate limiting, logging policy, or production hardening
for a real deployment.
